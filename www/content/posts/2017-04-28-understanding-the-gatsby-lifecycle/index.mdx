            rel="noopener"
            data-a11y="false"
            aria-label={`Link to ${option.name}`}
            href={option.url}
          >
            <Icon fill={fill} />
          </SocialIconContainer>
        );
      })}
    </>
  );
}

## Як працює Гетсбі?

Ми дивімось, що ми ці файли, ми повинні спершу виробити, як це працює Гетсбі. Гетсбі - це статистичний генератор веб-сайтів, який представив надійне джерело та створив веб-сайт / додав для вас.

Оновлена ​​версія встановленої версії для завантаження користувачів Bootstrap і Build. Під кришкою, веб-пакет використовує веб-пакет для створення та запуску сервера, який розкриває середні речі.

### Крок 1

Під час розширення Bootstrap, який ви робите, ви використовуєте $ gatsby розробити, зайнявшись 20 разів, відкривши для перевірки ваш gatsby-config.js, який цікавить інформацію про сторінку та сторінку для веб-сайту. Наприклад, перевірити Bootstrap - це те, що створюються сторінки. Якщо ви шукаєте глибоко з 20 кроків Bootstrap, Swyx діліться фантастичним принципом, який детальніше розглядає.

### Крок 2

Послідовні побудови дуже схожі на велику висоту завантаження програм Bootstrap, щоб зрозуміти, що вона працює з оптимізацією роботи та вводить статистичні файли, готові до розгортання. Підтримай про те, як звичайний варіант Реагуйте в режимі розвитку та розвитку.

### Крок 3

Після цього, після розкриття створених файлів, Гетсбі живе у браузері. Гетсбій створює статистичний веб-сайт, який після початкового завантаження перетворюється на веб-додаток, який робить більшу висоту циклу для браузера.

Ви можете пам’ятати, що можливий цикл Гетсбі може бути доступний у 3 основних рівнях:

* Завантажувач
* Побудуємо
* Веб-переглядач
* Ці три надійності складають великий цикл Гетсбі.

Частичне різноманітне цикл знайдеться із-за запущеного $ гецьби зараз Так само, як власник циклу React, Гетсбі розкривають гачки для виробників, над тим, що буде. Доступний для різних циклів можливий доступ до спеціальних файлів Gatsby, таких як gatsby-node.js, gatsby-browser.js та gatsby-ssr.js.

Для чого потрібні файли Гетсбі? gatsby-config.js Місце для розміщення всіх конфігурацій вашого веб-сайту, як плагіни, метадані та поліфайли. Цей файл є основним у ваших програмах, які знають у самому Там Гетсбі Попередньо проводять свою систему системних плагінів. Коли ви використовуєте $ gatsby development або $ gatsby build gatsby-config.js, цей перший файл, який потрібно прочитати та перевірити.

Більше Попереднього вашого часу, проводячи в gatsby-config.js, буде, звичайно, буде оновлено всі джерела плагінів, вигляд плагінів, офлайн-службовців, які створюють стилізацію та метаданих сайтів.

gatsby-node.js Gatsby використовує процес Node, коли ви розробляєте веб-сайт, або використовуйте веб-сайт, використовуючи Webpack під кришкою, щоб використовувати сервер розривки з поточного переходу. Під час роботи «Вузол» Гетсбі завантажує багато платних, перевірених мультфільмів, завантажує веб-сайт, будує схему даних, використовує сторінку та використовує роботу конфігурації та управління даними.

Все, що відбувається під час розширення Bootstrap і Build, відбувається в gatsby-node.js. Це означає, що це ідеальне місце для створення сторінки, яке динамічно базується на даних із вихідними плагінами, або змінює конфігурацію Webpack або Babel Gatsby.

Ви можете відкрити всі файли, які ви відкрили, але, файл Netlify _redirects, ви знайдете файли у своїх інших файлах, які надсилаєте gatsby-node.js, щоб знайти найпростіший цикл onPostBuild.

З точки зору, більше часу в даний час розглядався питання, який був створений і створений на сторінці gatsby-node.js. Цей файл швидко переходить на весь ваш веб-сайт.

## Приклади гачків gatsby-ssr.js:

* onPreRenderHTML

* ЗамінитиРендерер
* gatsby-browser.js

Gatsby is a static site that loads a dynamic application after initial load, which means you get the benefits of a static site in a web application. gatsby-browser.js provides convenient hooks to deal with app loading, route updates, service worker updates, scroll positioning, and more.

Everything that occurs after your static site has loaded can be hooked in gatsby-browser.js. For apps that I’ve built, gatsby-browser.js was mostly used for keeping track of routes, scroll positioning, and sending analytics events.

## Examples of gatsby-browser.js hooks:

* onClientEntry
* onRouteUpdate
* onServiceWorkerInstalled
* registerServiceWorker
* shouldUpdateScroll

## Conclusion

Gatsby is built with React at its core and shares a common API pattern, the lifecycle. This lifecycle gives developers access to key moments in their website’s process through specific hooks. For example, adding analytics can be achieved through the Browser lifecycle hook onClientEntry. Gatsby reserves specific filenames as an entry point to access every lifecycle; these files are named gatsby-node.js, gatsby-ssr.js and gatsby-browser.js.

Without the Gatsby lifecycle, it would be impossible to customize and modify your project beyond the base configuration, leaving developers with a rigid and poor developer experience. This power and flexibility has helped us build amazing web projects for clients like Hopper!
